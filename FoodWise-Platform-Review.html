FoodWise Platform

Full-Stack Restaurant Operations Management Platform
Technical Review & Feature Overview — March 2026
FoodWise is an all-in-one platform for food service operators to manage inventory, reduce waste, control costs, schedule staff, and make data-driven decisions. It runs on iOS, Android, and Web from a single codebase, backed by a fully serverless AWS infrastructure. The platform includes AI-powered forecasting, real-time POS integrations, and a comprehensive health scoring engine built on industry-standard restaurant KPIs.

Feature Overview

Core Operations

  • Real-time inventory tracking with barcode scanning
  • Low-stock alerts and expiration tracking (FIFO)
  • Waste logging by reason with cost analysis
  • Purchase orders with supplier email delivery
  • Shipment receiving with variance tracking
  • Physical inventory count sheets and count variance reports

Financial Intelligence

  • Weighted health score engine (food cost, waste, forecast, turnover, stockouts, labor)
  • Owner dashboard with multi-store comparison
  • 4 report types: Labor, P&L, Food Cost, Count Variance
  • Menu engineering with recipe costing and margin analysis
  • Vendor price history and trend tracking

Staff & Labor

  • Team management: invite, assign roles, deactivate
  • Role-based access (owner / manager / staff)
  • Shift scheduling with weekly view
  • Kiosk mode for employee clock-in/out with PIN auth
  • Timesheet approval and labor cost tracking
  • AI-powered weekly planner for staffing and ordering

AI & Forecasting

  • Claude-powered AI assistant (Amazon Bedrock) with store context
  • Python ML demand forecasting model (Docker Lambda)
  • Weekly forecast accuracy tracking
  • Forecast-driven prep lists
  • Recipe scaling calculator
  • Actionable health score recommendations

POS Integrations

  • Toast: HMAC-SHA256 webhook for real-time sync
  • Square: OAuth + EventBridge polling (15 min)
  • CSV: S3-triggered import or direct API upload
  • Fuzzy ingredient matching (Levenshtein distance)
  • Auto-mapping at 85% confidence, review at 60%
  • Transaction normalization and deduplication

Security & Compliance

  • Camera integration with footage viewing
  • Incident management and tracking
  • Temperature logging for food safety
  • Full audit trail (who changed what, when)
  • Push notifications for alerts and reminders
  • Offline mode with local data caching

Architecture

Client Layer
  React Native (Expo) → iOS + Android + Web

API Layer
  API Gateway → 14 Lambda Routers (Node.js 22, TypeScript)
  Cognito JWT Authentication on all endpoints

Data Layer
  25 DynamoDB Tables (PAY_PER_REQUEST, PITR on critical tables)

AI Layer
  Amazon Bedrock (Claude Sonnet) + Python ML Forecast (Docker Lambda)

Integration Layer
  Toast Webhooks | Square OAuth + Polling | CSV/S3 Import

Infrastructure
  AWS CDK → 5 Nested Stacks → ~324 CloudFormation Resources

Assessment

Strengths

  • Comprehensive scope covering the full restaurant operations lifecycle
  • Real financial metrics using industry-standard KPIs (food cost %, waste %, labor %)
  • Fully serverless — scales to zero, pay-per-use, zero servers to manage
  • Single codebase for three platforms (iOS, Android, Web)
  • AI-native design with assistant and forecasting built in
  • Production-grade infrastructure with PITR, monitoring, and security hardening

Growth Opportunities

  • Connect live POS data from a real store environment
  • Add unit and integration test suite
  • Set up CI/CD pipeline for automated deployments
  • Add stricter API input validation (e.g., Zod schemas)
  • Implement offline conflict resolution for sync
  • Expand to multi-tenant SaaS with billing
